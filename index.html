üß† Project Overview

CrowdCount is a real-time AI-powered crowd analytics system designed to detect, track, and count people across multiple surveillance zones using computer vision. It enables administrators and security operators to monitor crowd density, set zone thresholds, and get instant alerts when thresholds are exceeded ‚Äî all through an elegant Flask-based web dashboard.

üß© Technologies & Modules Used

Flask

Backend web framework for handling routes, templates, and APIs

SQLite3

Lightweight database for storing users, zones, logs, and camera info

Werkzeug Security

Password hashing (generate_password_hash, check_password_hash) for secure login

JWT (PyJWT)

To generate JSON Web Tokens for session security

OS, Datetime, Time, IO, CSV

For file management, timestamping logs, and exporting CSV reports

OpenCV (cv2)

To handle video input, frame processing, and visualization

Ultralytics YOLOv8

For person detection using pre-trained deep learning models

DeepSORT

For object tracking across frames, assigning consistent IDs to objects

Threading

For running video capture and tracking asynchronously without freezing the Flask app

ReportLab

For generating PDF exports of logs directly from the backend

Jinja2 Templates

HTML rendering for dashboard, admin panel, etc

JWT (JSON Web Token)

Secure authentication across user sessions

üß± Functional Breakdown

init_db()

Creates all the necessary tables (users, zones, user_profiles, activity_logs, cameras, etc.). Ensures schema compatibility for future upgrades.

allowed_file(filename)

Validates uploaded video files based on allowed extensions.

log_activity(...)

Inserts an activity record into activity_logs with user actions and timestamps. Used for tracking everything.

is_admin(user_id)

Checks if a user has an admin role in the database.

admin_required(f)

Decorator that restricts certain routes to admin users only. If not admin, redirects to dashboard.

/zm_start_tracking

Starts YOLO+DeepSORT tracking for an uploaded video.

/zm_stop_tracking

Stops the ongoing tracking thread.

/zm_feed

Provides real-time MJPEG streaming output of the tracker.

/start_stream

Starts live streaming from webcam/IP camera.

/video_feed

Live video feed endpoint for dashboard.

/stats_stream

Sends live JSON event updates (counts, thresholds) using SSE (Server-Sent Events).

/register

Handles new user registration with password hashing and logging.

/login

Authenticates user and generates JWT token.

/logout

Ends user session and logs logout activity.

/profile

Lets user view and update personal profile info (dob, gender, etc.).

/admin

Main admin dashboard showing overall system stats.

/admin/users

Displays all users for management.

/admin/activity-log

Paginated view of all activity logs.

/admin/export-logs/csv

Exports full activity logs as CSV.

/admin/export-logs/pdf

Exports activity logs as PDF using ReportLab.
